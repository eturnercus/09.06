# WatchAlert

Кроссплатформенное приложение для **Linux** и **Windows**, которое отслеживает выбранные области экрана и воспроизводит звук, если изменение сохраняется заданное количество секунд.

## Возможности

- Несколько независимых областей мониторинга
- Настраиваемая задержка перед сигналом (1–3600 сек)
- Любой звуковой файл (WAV, MP3, OGG, FLAC)
- Настройка чувствительности к изменениям
- Живое превью каждой области
- Автосохранение настроек в `~/.watchalert/config.json`

## Требования

- Python 3.10+
- Linux или Windows

На Linux:

```bash
sudo apt install python3-tk libsdl2-dev scrot
```

**Захват экрана на Linux** — приложение **само подбирает** рабочий способ при старте:

| Среда | Способы (по порядку) |
|-------|----------------------|
| Wayland (GNOME/KDE) | XDG Portal, grim, spectacle, gnome-screenshot, Pillow XCB |
| X11 | Pillow XCB, mss xlib, scrot, maim, ImageMagick, ffmpeg |

Кнопка **«Проверить захват»** в настройках показывает, что работает у вас.

**Не запускайте через sudo.** Запуск: `./WatchAlert.AppImage`

Если не работает на Wayland: `sudo apt install grim scrot` или войдите в сессию **Ubuntu on Xorg**.
Принудительно: `WATCHALERT_CAPTURE=pillow_x11 ./WatchAlert.AppImage`

На Windows обычно достаточно установить Python с [python.org](https://www.python.org/) (включите «tcl/tk» при установке).

## Как пользоваться

1. Укажите **задержку** — сколько секунд изменение должно сохраняться до сигнала.
2. Выберите **звуковой файл** и нажмите «Тест» для проверки.
3. Нажмите **«Добавить область»** — на **каждом мониторе** появится замороженный снимок этого монитора. Выделите мышью прямоугольник на том экране, где нужно следить.
4. Повторите для других областей при необходимости.
5. Нажмите **«▶ Старт»** — мониторинг начнётся, для каждой области откроется окно превью.
6. Если содержимое области изменится и останется изменённым заданное время — прозвучит сигнал.

**Esc** — отмена выбора области.

## Принцип работы

Приложение периодически снимает скриншот каждой области **в память** (файлы на диск не создаются) и сравнивает с одним эталонным кадром. Старый кадр заменяется новым — мусора на диске нет, в RAM хранится только текущий эталон на область. При обнаружении устойчивого отличия запускается таймер. Если отличие исчезает — таймер сбрасывается. По истечении заданной задержки воспроизводится звук, и эталон обновляется (чтобы не срабатывать повторно на том же изменении).

## Длительная работа

Приложение рассчитано на **непрерывную работу** (24/7):

- фоновый поток на каждую область работает бесконечно, пока не нажмёте «Стоп»;
- каждый захваченный кадр освобождается сразу после обработки;
- превью обновляется с прореживанием (в очереди GUI всегда только последний кадр);
- при временных сбоях захвата экрана поток не падает, а делает паузу и продолжает.

## Расширение для браузера

**WatchAlert Tab** — тот же принцип, но зоны на **вкладках** браузера (мониторинг в фоне, пока вы в других вкладках). Поддерживаются **Chrome, Edge и Firefox**.

Скачайте zip из [Releases](https://github.com/eturnercus/09.06/releases) или соберите: `./build/build-extension.sh`

Установка из исходников: папка [`extension/`](extension/) → `chrome://extensions/` или Firefox `about:debugging`.  
Подробнее: [extension/README.md](extension/README.md).
